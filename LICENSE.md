## ❌ Troubleshooting Common Issues

- ❌ **Bytebreaker Executor fails to attach**
  The client cannot find the game process, usually because it launched before the game fully loaded.
  **Solution:** Close **Bytebreaker Executor**, reopen the game, wait until you reach the main menu, then relaunch and click `Attach`.

- ❌ **Antivirus quarantines the installer**
  Some security software flags injection tools as potentially unwanted programs.
  **Solution:** Add the entire `C:\...\BytebreakerExecutor\` folder to your antivirus exclusion list before extracting or installing.

- ❌ **Scripts execute but nothing happens in-game**
  The script may target a different game or rely on APIs not present in the current title.
  **Solution:** Verify script compatibility, check the **Bytebreaker Executor** debug console for error output, and confirm you are in the correct game.

- ❌ **UI appears blurry or oversized on high-DPI screens**
  Windows display scaling can distort the interface at settings above 100%.
  **Solution:** Right-click `bytebreaker.exe` → `Properties` → `Compatibility` → `Change high DPI settings` → enable `Override high DPI scaling`.

- ❌ **Auto-attach does not trigger on game launch**
  The `auto_attach` setting may be disabled or the `injection_delay` is too low.
  **Solution:** Open `config.json`, set `"auto_attach": true` and increase `"injection_delay"` to `2500`, then restart **Bytebreaker Executor**.

- ❌ **Cloud sync shows "authentication failed"**
  Your session token has expired or the sync server is temporarily offline.
  **Solution:** Log out and log back in through the **Bytebreaker Executor** settings panel; if the issue persists, check the status page.
